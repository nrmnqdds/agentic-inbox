# Dark Mode Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a persisted light/dark theme toggle to Agentic Inbox, applied via kumo's built-in `[data-mode]` tokens, with no first-paint flash.

**Architecture:** A small zustand store owns theme state and persists to localStorage; an inline anti-flash script in `root.tsx` applies the saved theme to `<html data-mode>` before first paint; a new "Appearance" card on the settings page toggles instantly. Dark styling itself is free — kumo overrides every semantic token under `[data-mode="dark"]`.

**Tech Stack:** React Router 7 (SSR), Tailwind CSS v4, `@cloudflare/kumo`, zustand, TypeScript.

### Task 1: Theme store

**Files:**
- Create: `app/hooks/useThemeStore.ts`
- Reference (style only): `app/hooks/useUIStore.ts`

**Interfaces:**
- Consumes: `create` from `zustand` (already a dependency, used by `useUIStore`).
- Produces (used by Task 3):
  - `export type Theme = "light" | "dark"`
  - `export const THEME_STORAGE_KEY: "agentic-inbox-theme"`
  - `export const useThemeStore: UseBoundStore<StoreApi<ThemeState>>` where `ThemeState` is `{ theme: Theme; setTheme: (theme: Theme) => void; toggleTheme: () => void }`
  - `setTheme` side effects: writes `THEME_STORAGE_KEY` to localStorage (guarded), sets `document.documentElement.dataset.mode = theme` and `document.documentElement.style.colorScheme = theme`, updates store state.

- [ ] **Step 1: Create the store**

Create `app/hooks/useThemeStore.ts` with exactly this content:

```ts
// Copyright (c) 2026 Cloudflare, Inc.
// Licensed under the Apache 2.0 license found in the LICENSE file or at:
//     https://opensource.org/licenses/Apache-2.0

import { create } from "zustand";

export type Theme = "light" | "dark";

/**
 * localStorage key for the theme preference.
 * NOTE: This key string is duplicated in the inline anti-flash script in
 * app/root.tsx — that script runs before app modules load and cannot import
 * this constant. Keep both in sync.
 */
export const THEME_STORAGE_KEY = "agentic-inbox-theme";

function isTheme(value: unknown): value is Theme {
	return value === "light" || value === "dark";
}

/**
 * Read the persisted theme. Returns "light" when unset, invalid, or storage
 * is unavailable (e.g. private browsing). SSR-safe.
 */
function readStoredTheme(): Theme {
	if (typeof window === "undefined") return "light";
	try {
		const stored = window.localStorage.getItem(THEME_STORAGE_KEY);
		return isTheme(stored) ? stored : "light";
	} catch {
		return "light";
	}
}

/**
 * Apply a theme to the document root. Kumo's semantic tokens flip under
 * [data-mode="dark"]; color-scheme makes native scrollbars and form
 * controls follow.
 */
function applyThemeToDocument(theme: Theme) {
	const root = document.documentElement;
	root.dataset.mode = theme;
	root.style.colorScheme = theme;
}

interface ThemeState {
	theme: Theme;
	setTheme: (theme: Theme) => void;
	toggleTheme: () => void;
}

export const useThemeStore = create<ThemeState>((set, get) => ({
	theme: readStoredTheme(),

	setTheme: (theme) => {
		try {
			window.localStorage.setItem(THEME_STORAGE_KEY, theme);
		} catch {
			// Storage unavailable — theme still applies for this session.
		}
		applyThemeToDocument(theme);
		set({ theme });
	},

	toggleTheme: () => {
		get().setTheme(get().theme === "dark" ? "light" : "dark");
	},
}));
```
